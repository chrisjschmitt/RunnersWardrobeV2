Part 2: Implementation Steps

Phase 1: Payment Infrastructure (PWA)

‚è±Ô∏è Estimated time: 2-3 days

Step 1: Set Up Stripe Account

  1. Go to stripe.com and create an account
  2. Complete business verification
  3. Get API keys (test keys first, then live)
  4. Set up products: Monthly ($2.99) and Annual ($19.99)

Step 2: Create Backend

api/ ‚îú‚îÄ‚îÄ create-checkout.ts # Creates Stripe checkout session ‚îú‚îÄ‚îÄ webhook.ts # Handles Stripe payment events ‚îî‚îÄ‚îÄ verify-license.ts # Checks if user has active subscription

Step 3: User Identification

  1. On first premium prompt, ask for email
  2. Store email in IndexedDB
  3. Email becomes the license key
  4. Stripe checkout pre-fills this email

Step 4: Frontend Gating

  1. Add isPremium flag to AppSettings
  2. Create <PremiumGate> component
  3. Add upgrade prompts at gate points
  4. Create <UpgradeModal> with Stripe redirect

Phase 2: Google Play Store

‚è±Ô∏è Estimated time: 1-2 hours + 3-7 days review

Step 1: Generate Android App

  1. Go to pwabuilder.com
  2. Enter your deployed PWA URL
  3. Click "Package for stores" ‚Üí Android
  4. Download the generated APK/AAB bundle

Step 2: Create Developer Account

  1. Go to play.google.com/console
  2. Pay $25 one-time registration fee
  3. Complete identity verification

Step 3: Create Listing & Submit

  1. App name, description, screenshots
  2. Upload AAB bundle
  3. Set content rating and pricing
  4. Add privacy policy URL
  5. Submit for review (3-7 days)

Phase 3: Apple App Store

‚è±Ô∏è Estimated time: 1-2 days + 1-2 weeks review

Step 1: Apple Developer Account

  1. Go to developer.apple.com
  2. Enroll as individual ($99/year)
  3. Complete identity verification (24-48 hours)

Step 2: Install Capacitor

# Install Capacitor npm install @capacitor/core @capacitor/cli npx cap init "Outdoor Wardrobe" "com.outdoorwardrobe.app" # Add iOS platform npm install @capacitor/ios npx cap add ios # Build and copy npm run build npx cap copy ios

Step 3: Configure & Submit

  1. Open in Xcode: npx cap open ios
  2. Configure signing with Apple Developer account
  3. Add privacy descriptions in Info.plist
  4. Create App Store listing with screenshots
  5. Archive and upload via Xcode
  6. Submit for review (1-2 weeks)

Part 3: Privacy Policy Requirements

Both app stores require a privacy policy covering:

  1. Data collected: Location (for weather), Activity history, Email (if using for license)
  2. How data is used: Location for weather only, History stored on device only
  3. Third parties: OpenWeatherMap, Stripe (if payments), Vercel
  4. Data retention: All data stored locally, user can delete anytime
  5. Contact info: Your email for privacy inquiries
üí° Tip: Use a privacy policy generator like Termly or GetTerms.io
